Model Overview

Goal: Demonstrate explainable risk scoring on synthetic identity/transaction data with auditable lineage.

  • Relational core: Customer, Account, Device, LoginEvent, Txn
  • Feature views: velocity (30m), geo/device mismatch, device reputation
  • Materialized risk: fact_risk (rule score + optional model score)
  • Power BI: overview KPIs, analyst workbench, customer 360, rules monitoring

All data are synthetic. Any sensitive fields are hashed or simulated.

Architecture

  • Lineage columns on every table (record_src, batch_id, created_at)
  • Hash-based surrogate (person_hash) to keep PII out of analytics
  • Star schema exposures for BI; RLS and incremental refresh ready

Explainability

  • Rule hits (velocity, geo mismatch, device rep) shown per transaction
  • Optional model score; combine as risk_score = rule + model
  • Closed case outcomes flow back to label store (continuous learning)
